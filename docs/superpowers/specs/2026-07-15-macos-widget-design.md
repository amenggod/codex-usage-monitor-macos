# Codex Usage Monitor 原生 macOS 桌面小组件设计

日期：2026-07-15
状态：设计已口头确认，等待书面复核
目标平台：macOS 14 及以上

## 1. 背景

现有“桌面卡片”由应用进程创建无边框 `NSPanel`。它可以模拟桌面卡片，但不是 macOS 系统小组件，无法进入系统小组件图库，也不能获得与时钟、日历和天气相同的固定、移动和尺寸管理体验。当前实现还在应用完成启动前调用窗口显示，导致安装版本出现“进程已经运行但窗口为 0”的启动故障。

用户确认需要真正的 macOS WidgetKit 小组件：由系统固定在桌面，外观接近系统时钟和天气卡片，点击后打开 Codex Usage Monitor 完整窗口。用户接受 WidgetKit 的刷新由系统调度，桌面数据可能晚于主程序的实时数据。

## 2. 目标

- 提供可从 macOS 小组件图库添加到桌面的原生 WidgetKit 小组件。
- 支持系统小号和中号两种尺寸。
- 小组件显示最近一次可信 Token 统计、有效限额、项目摘要和更新时间。
- 点击小组件打开或前置应用的完整统计窗口。
- 后台主程序继续近实时读取本地 Codex 日志、计算统计并执行阈值提醒。
- 主程序与小组件通过最小化、脱敏的共享快照交换数据。
- 5 小时限额缺失或过期时完全隐藏，不显示空占位，也不触发提醒。
- 主程序未运行或共享数据过期时保留最后可信值，并明确显示更新时间或过期状态，不回退为误导性的 0。
- 保留用户对顶部菜单栏入口和登录启动的控制。
- 为 Developer ID 签名、公证和公开仓库构建提供可复现结构。

## 3. 非目标

- 不承诺桌面小组件秒级刷新。WidgetKit 拥有最终刷新调度权。
- 不让 Widget Extension 直接扫描 `~/.codex`、维护 SQLite 索引或发送限额通知。
- 不读取、保存或共享提示词、回复、工具输出、凭据或未经脱敏的完整项目路径。
- 不继续维护伪装成系统小组件的 `NSPanel` 桌面卡片。
- 不在本阶段增加云同步、团队看板、费用估算或多设备聚合。
- 不尝试由应用自动添加、移动或删除系统小组件；这些操作由 macOS 管理。

## 4. 方案选择

### 4.1 仅 Widget Extension 读取日志

该方案组件少，但 Widget Extension 不持续运行，受刷新预算和文件访问限制，无法满足尽可能及时、稳定的统计要求，不采用。

### 4.2 继续使用桌面 `NSPanel`

该方案可以秒级更新，但不是系统小组件，不符合已确认的桌面交互，不采用。

### 4.3 主程序监控 + WidgetKit 展示

采用该方案。主程序作为权威数据生产者，继续处理日志、索引、限额和通知；Widget Extension 只读取共享快照并渲染。数据变化时主程序请求 WidgetKit 重新加载时间线，同时保留系统调度限制。

## 5. 工程与目标结构

现有仓库是 Swift Package 加脚本生成 `.app`，没有 Xcode 工程。原生 Widget Extension 必须成为应用包内的扩展目标，因此新增并提交 Xcode 工程，至少包含：

- `CodexUsageMonitorApp`：主应用目标；
- `CodexUsageMonitorWidget`：WidgetKit Extension 目标；
- `CodexUsageMonitorLoginItem`：只负责登录时以后台参数启动主应用并立即退出的轻量登录项目标；
- `CodexUsageShared`：主应用和扩展共同依赖的轻量共享模块；
- 现有测试目标，以及新增共享快照和小组件策略测试。

现有日志解析、SQLite、聚合、通知和完整界面继续属于主应用或其核心模块。共享模块只包含可以安全跨进程使用的值类型、编解码和共享存储接口，不依赖 SQLite、日志扫描器、通知服务或应用窗口。

Swift Package 继续作为核心逻辑测试入口。Xcode 工程负责应用、Extension、能力、签名和最终归档。构建脚本改用 `xcodebuild` 生成包含 Widget Extension 的应用包；只复制 SwiftPM 可执行文件的旧打包路径停止作为正式发布方式。

## 6. 标识、能力与签名

默认标识为：

- 主应用：`com.amenggod.CodexUsageMonitor`；
- Widget Extension：`com.amenggod.CodexUsageMonitor.Widget`；
- App Group：`group.com.amenggod.CodexUsageMonitor`；
- 小组件 kind：`com.amenggod.CodexUsageMonitor.usage`；
- 深度链接：`codexusagemonitor://dashboard`。

主应用与 Widget Extension 都启用同一个 App Group。正式安装包需要为主应用和内嵌 Extension 使用一致团队签名并带正确 entitlement。公开仓库只保存工程配置和 entitlement 模板，不保存证书、私钥、Apple ID、App Store Connect 密钥或公证凭据。

无签名 CI 可以执行逻辑测试和 `CODE_SIGNING_ALLOWED=NO` 编译检查。要让 macOS 在小组件图库中稳定发现扩展，并完成 Developer ID 分发与公证，发布阶段必须使用有效 Apple Developer 签名。

## 7. 共享快照

### 7.1 快照内容

`WidgetUsageSnapshot` 是带版本号的 `Codable` 值，包含：

- `schemaVersion`；
- `generatedAt`；
- 数据可信状态；
- 今日 Token；
- 全部历史 Token；
- 可选的有效 5 小时限额；
- 可选的有效周限额；
- 最多三个项目的显示名和 Token；
- 可选的失败文件数量或重建状态摘要。

限额只保存展示所需的剩余比例、重置时间和稳定限额标识。项目只保存已经脱敏的显示名，不保存完整路径。共享快照不包含单条事件、会话正文、文件偏移、日志路径或通知回执。

### 7.2 写入规则

主程序每次发布新的可信聚合快照时生成小组件快照。写入采用同目录临时文件加原子替换，保证 Widget Extension 不会读取半份 JSON。只有小组件实际展示字段发生变化，或更新时间跨过需要显示的新鲜度边界时，才调用 `WidgetCenter.reloadTimelines(ofKind:)`，避免无意义消耗刷新预算。

写入失败不影响主程序内的实时统计。主程序记录脱敏错误状态，并在下一次数据变化或定时恢复检查时重试。

### 7.3 读取规则

Widget Extension 从 App Group 容器读取快照：

- 没有快照：显示“打开 Codex Usage Monitor 完成首次同步”；
- 快照有效：显示统计和更新时间；
- 快照过期：保留最后可信统计并显示“上次更新”；
- 快照版本未知或损坏：显示“等待重新同步”，不显示 0；
- 5 小时限额不存在或已经过期：不渲染对应内容；
- 周限额不存在或已经过期：显示“等待周限额”，不沿用旧剩余比例。

## 8. 小组件界面

### 8.1 视觉语言

小组件使用系统圆角容器、系统材质和语义颜色，跟随浅色、深色以及 macOS 小组件着色模式。信息密度参考用户提供的时钟、天气和电量小组件：大数字、短标签、环形或条形进度、充足留白，不复制天气图标或系统品牌元素。

WidgetKit 负责外框、尺寸和桌面位置。界面不绘制窗口标题栏、关闭按钮、拖动区域或自定义阴影。

### 8.2 小号 `systemSmall`

小号以今日用量为视觉中心：

- 标题 `Codex`；
- 今日 Token 大数字；
- 有效周限额的剩余环形进度；
- 简短更新时间或过期标记。

若周限额不可用，环形区域改为“等待限额”，今日 Token 仍正常展示。小号不显示项目排行，避免拥挤。

### 8.3 中号 `systemMedium`

中号分为摘要和项目两部分：

- 今日 Token；
- 全部 Token；
- 有效周限额进度和重置摘要；
- 有效 5 小时限额存在时显示其进度；
- 最多三个项目的 Token 排行；
- 最近更新时间或数据状态。

5 小时限额隐藏后，周限额和项目区自动扩展，不保留空槽位。项目少于三个时只显示实际项目。

### 8.4 点击行为

小组件根视图设置 `widgetURL(codexusagemonitor://dashboard)`。点击任意非交互区域由系统启动主应用。主应用接收 URL 后创建或前置唯一的完整统计窗口；重复点击不会创建多个相同窗口。

## 9. 主应用窗口与运行方式

新增唯一标识的完整统计 `WindowGroup`，复用现有今日、7 天、全部、项目排行、数据状态、重试和设置能力。以下操作都必须打开或前置该窗口：

- 点击系统小组件；
- 在 Finder 中双击应用；
- 应用已经运行时再次打开；
- 点击菜单栏面板中的“打开完整统计”。

登录启动由独立 `CodexUsageMonitorLoginItem` 完成。登录项使用不激活前台的启动配置，并向主应用传入 `--background`；主应用收到该参数后启动监控但不打开完整窗口，登录项随后退出。Finder 正常启动不带该参数，因此首次手动启动立即打开完整统计窗口。深度链接和应用再次打开事件无论当前是否为后台状态，都会打开或前置完整窗口。

设置中的“登录时启动”改为注册或注销这个登录项，而不是直接注册主应用。升级时先清理旧的主应用登录注册，再按用户原有选择注册新登录项。用户主动退出主应用后，小组件继续显示最后可信快照并标注更新时间；不会伪装成仍在实时更新。

旧 `NSPanel` 桌面卡片及其位置、展开状态和三态 `DisplayMode` 停止使用。迁移规则为：

- 旧模式为 `menuBar` 或 `both`：保留菜单栏入口；
- 旧模式为 `desktop`：默认关闭菜单栏入口；
- 旧桌面卡片位置和展开键不再读取，但无需主动删除；
- 系统小组件是否存在由 macOS 小组件配置决定，不写入应用设置。

设置页把“显示位置”替换为“显示顶部菜单栏”开关，并增加只读的“如何添加桌面小组件”说明：在桌面空白处右键，选择编辑小组件，然后搜索 Codex Usage Monitor。

## 10. 实时性与刷新策略

主应用继续使用文件监听，在 Codex 日志写入后尽快更新内部统计。主界面和菜单栏仍以主应用内状态为准，可以近实时更新。

小组件采用事件驱动加保底时间线：

- 展示字段变化时，主应用请求重新加载指定 kind；
- Widget Provider 每次读取最新共享快照生成时间线；
- 时间线提供较低频保底刷新，用于更新时间、过期状态和重置边界；
- 不创建秒级或分钟级密集时间线；
- 小组件始终显示最后更新时间，使用户能区分实时主界面和系统调度后的桌面结果。

## 11. 错误处理

- App Group 容器不可用：主应用继续工作，在设置页显示“小组件共享不可用”；Widget 使用占位状态。
- 快照写入失败：保留旧快照，稍后重试，不覆盖为零值。
- 快照读取或解码失败：Widget 显示等待同步，不崩溃。
- 主程序数据部分失败：快照带部分失败状态，Widget 显示简短警示，已成功统计仍可见。
- 主程序正在重建：Widget 保留最后可信结果，并显示重建状态。
- 深度链接无法解析：主应用回退到打开默认完整统计窗口。
- Widget Extension 崩溃或超时不影响主应用监控和通知。

## 12. 隐私与安全

- Widget Extension 不访问 Codex 日志目录。
- App Group 只存展示快照，不存提示词、回复、工具调用正文或凭据。
- 项目名称沿用当前脱敏显示规则，不把完整路径写入共享容器。
- 日志和错误不输出共享快照正文或用户路径。
- entitlement 采用最小能力，仅添加 Widget 和 App Group 所需配置。

## 13. 测试设计

### 13.1 共享数据测试

- 快照编码、解码和 schema 版本兼容；
- 原子写入后只能读到旧完整快照或新完整快照；
- 未知版本、损坏文件和缺失文件得到明确占位状态；
- 共享快照不包含完整项目路径、事件正文或凭据字段；
- 展示字段未变化时不重复请求刷新；
- 展示字段变化、限额失效和重置边界会请求刷新。

### 13.2 展示策略测试

- 小号显示今日 Token、周限额和更新时间；
- 中号显示今日、全部、项目排行和有效限额；
- 缺失或过期 5 小时限额时完全隐藏且布局重排；
- 过期周限额不显示旧剩余比例；
- 无快照、过期快照、部分失败和重建状态有可理解文案；
- Token 大数值使用稳定紧凑格式，不溢出两种尺寸。

### 13.3 应用交互测试

- `codexusagemonitor://dashboard` 打开唯一完整统计窗口；
- Finder 首次打开和再次打开都显示完整统计窗口；
- 登录项以 `--background` 启动时不打开窗口，主程序仍开始监控；
- 菜单栏入口开关即时生效并持久化；
- 旧 `DisplayMode` 正确迁移为菜单栏布尔设置；
- 登录启动可以在没有前台窗口时继续产生共享快照；
- 用户退出后 Widget 显示最后更新时间而非实时状态。

### 13.4 构建与安装验证

- `swift test` 全部通过；
- 主应用和 Widget Extension 在无签名 CI 中编译通过；
- 使用 Apple Development 签名的本地包能在 macOS 小组件图库中找到；
- 小号和中号均可添加、移动、删除；
- 点击真实桌面小组件能打开或前置完整统计窗口；
- Developer ID 归档中主应用和内嵌 Extension 的签名、entitlement 和 App Group 一致；
- 公证上传、staple 和 Gatekeeper 验证通过后才标记正式发布包。

## 15. 验收标准

- macOS 小组件图库可以找到 Codex Usage Monitor。
- 用户能把小号或中号小组件固定到桌面，行为与系统时钟、天气类小组件一致。
- 小组件显示最近一次可信统计和明确更新时间，缺失数据不显示为误导性的 0。
- 5 小时限额不可用时完全隐藏；恢复有效数据后自动重新出现。
- 主程序运行时继续近实时更新内部统计、项目排行、限额和通知，并请求刷新桌面小组件。
- 点击小组件、双击应用或使用菜单栏入口都能打开同一个完整统计窗口。
- 旧伪桌面卡片不再出现，启动后不会存在“进程运行但没有任何可达入口”的状态。
- 共享容器中不存在提示词、回复、工具输出、凭据或完整项目路径。
- 签名版本能通过 Extension、App Group、Developer ID 和公证验证。
