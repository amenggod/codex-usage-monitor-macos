# 原生桌面小组件刷新预算设计

## 背景

Codex Usage Monitor 主程序每分钟读取一次实时额度，并把脱敏后的最新数据写入 App Group。故障现场显示：主程序和 App Group 快照都已更新到同一个周剩余百分比，但桌面小组件仍显示旧值。`chronod` 稍后才记录刷新请求，证明不一致发生在 WidgetKit 的时间线缓存层，而不是额度读取或共享文件层。

当前发布器把包含 `observedAt` 的实时额度新鲜度放进刷新指纹。即使可见数值完全没有变化，新的观测时间也会使指纹每分钟变化，进而反复调用 WidgetKit 重载。macOS 可以合并或限制这些请求；过度请求会浪费有限刷新机会，使真正的额度变化更容易延迟显示。

## 目标

- 保留原生 macOS WidgetKit 小组件及其系统桌面固定体验。
- 主程序继续每分钟更新 App Group 快照，作为唯一共享数据源。
- 小组件继续显示最近一次可信额度，并明确显示更新时间。
- 只有用户可见内容或数据状态真正变化时才主动请求 WidgetKit 重载。
- 额度变化后尽快刷新，但不承诺 WidgetKit 无法保证的秒级或严格分钟级重绘。
- 点击小组件继续打开完整主程序，供用户查看实时值和手动刷新。

## 非目标

- 不用浮动窗口或自绘桌面卡片替代 WidgetKit。
- 不尝试绕过 macOS 的小组件刷新预算。
- 不把凭据、提示词内容、完整路径或完整 app-server 响应写入共享快照。
- 不让程序与 Only Switch 等第三方菜单栏管理工具争夺状态项位置。

## 方案选择

采用“数据变化驱动重载 + 五分钟时间线兜底”。

主程序仍在每次成功聚合后写入完整快照，因此文件中的 `generatedAt`、`observedAt` 和状态时间保持最新。发布器另外构造只描述可见内容的刷新指纹：Token 总量、限额值与重置时间、项目排行、数据状态类别及错误文件数。新鲜度指纹只比较 `fresh`、`stale`、`unavailable` 类别，不比较每分钟变化的观测时间。

当可见指纹变化时，发布器调用 `reloadTimelines`；仅时间戳变化时只写快照，不主动消耗刷新请求。Widget 时间线继续每五分钟重新读取快照，并在更早的额度重置时刻刷新，从而更新“更新于”文案和过期状态。

## 数据流

1. 实时额度服务每分钟读取 Codex app-server。
2. `UsageViewModel` 合并 Token 统计与实时额度。
3. `WidgetSnapshotPublisher` 每次都把最新脱敏快照原子写入 App Group。
4. 发布器计算可见内容指纹：
   - 指纹改变：请求 WidgetKit 重载。
   - 仅观测时间改变：不请求重载。
5. `UsageTimelineProvider` 在系统允许时读取最新快照；默认五分钟后再次请求时间线。
6. 小组件显示最近有效值与明确的更新时间；点击后打开完整面板。

## 展示与陈旧状态

- 快照不超过 15 分钟：显示“更新于 HH:mm”。
- 快照超过 15 分钟：继续显示最近有效数值，并显示“上次更新 HH:mm”。
- 实时额度超过 10 分钟未观测：显示“上次实时同步 HH:mm”。
- 实时额度超过 30 分钟未观测：隐藏额度数值并显示“实时限额不可用”，避免把过旧额度伪装为当前值。
- 数据文件缺失、损坏或从未成功同步时，继续使用现有不可用状态文案。

这些规则保留当前安全边界：一般 Token 数据可以显示旧值并标注时间，但超过可信窗口的实时额度不会继续冒充当前额度。

## 菜单栏兼容性

菜单栏问题与小组件刷新是两个独立故障。现场确认 Only Switch 把状态项窗口移动到屏幕外；退出 Only Switch 并重新创建状态项后恢复。程序继续提供菜单栏开关和模板图像以适配深浅背景，但不主动对抗第三方菜单栏折叠工具。README 和设置说明应明确：需要 Codex 额度常驻可见时，不要让 Only Switch 折叠该状态项。

## 错误处理

- 快照写入失败：主程序显示“小组件共享不可用”，不影响主程序自身额度展示。
- WidgetKit 忽略或延迟重载：保留最近有效值和更新时间，等待下一次系统时间线机会。
- 聚合失败：保留最近可信数据并使用现有 `stale` 或 `failed` 状态。
- 并发发布：继续保留请求序号保护，较旧请求不能覆盖较新快照。

## 测试

- 两次可见数据相同、只有 `observedAt` 改变：写入两次快照，但只请求一次重载。
- 周剩余百分比改变：必须请求第二次重载。
- Token、项目排行、限额出现或消失、状态类别改变：必须请求重载。
- `fresh` 变为 `stale` 或 `unavailable`：必须请求重载。
- 五分钟时间线兜底、额度重置优先级和 15 分钟陈旧文案继续通过测试。
- 完整 Swift 测试、Xcode 构建、签名与已安装程序验证全部通过后才交付。

## 验收标准

- 主程序和 App Group 快照继续每分钟同步。
- 无可见变化时不再每分钟调用 WidgetKit 重载。
- 额度或 Token 真正变化时产生重载请求。
- 原生小组件始终展示最近有效值及其更新时间，或明确展示不可用状态。
- 菜单栏在未被第三方工具折叠时正常显示。
