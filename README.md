# Codex Usage Monitor

## 概览

Codex Usage Monitor 是一款常驻 macOS 菜单栏的本地用量查看工具。它从本机 Codex 会话日志中汇总 Token 用量与限额状态，不需要上传会话内容。

本项目是非官方开源项目，与 OpenAI 没有隶属关系，也未获得 OpenAI 的认可或背书。Codex 和 OpenAI 是其各自权利人的商标。

## 功能

- 在菜单栏显示 5 小时和每周限额的剩余百分比。
- 查看今日、最近 7 天和全部时间的 Token 总量。
- 按项目汇总并排序用量。
- 显示限额重置时间、数据新鲜度和读取失败状态。
- 增量监听当前与已归档的 Codex 会话日志。
- 支持重新扫描以及清空本地索引后重建。
- 可选的限额通知与登录时自动启动。

## 系统要求

- macOS 14 Sonoma 或更高版本。
- 从源码构建需要 Swift 6 工具链。
- 当前构建脚本生成当前 Mac 主机架构的应用，不是通用二进制。

## 隐私边界

应用只在本机读取 `CODEX_HOME` 指向的目录；未设置时使用 `~/.codex`。扫描范围仅包括：

- `sessions/**/*.jsonl`
- `archived_sessions/**/*.jsonl`

解析器只使用会话标识、工作目录、Token 计数、限额百分比和重置时间。提示词、回复正文、工具输出和凭据等内容不会被索引。汇总索引保存在用户的 Application Support 目录中，不会由本应用发送到网络。

工作目录用于项目归类；完整路径可能出现在界面帮助或辅助功能提示中，但不会写入通知正文。若不希望应用读取会话日志，请退出应用或不要授予相关文件访问权限。

## 构建

在仓库根目录运行测试：

```bash
swift test
```

构建 release 应用、进行 ad-hoc 签名并生成 ZIP：

```bash
bash Scripts/build-app.sh
```

产物位于：

- `dist/Codex Usage Monitor.app`
- `dist/Codex-Usage-Monitor-macOS.zip`

构建脚本会校验 `Info.plist` 和应用签名。ad-hoc 签名仅用于本地完整性，不等同于 Developer ID 签名或 Apple 公证。

## 安装

1. 解压 `dist/Codex-Usage-Monitor-macOS.zip`。
2. 将 `Codex Usage Monitor.app` 拖入“应用程序”文件夹。
3. 首次启动时，如果 macOS 对未公证应用给出提示，请在 Finder 中右键应用并选择“打开”，确认应用来自你信任的构建或源码。
4. 启动后在菜单栏中找到 Codex Usage Monitor。

发布仓库不会提交 `.app` 或 `.zip` 产物；请从可信发布附件获取，或按上述步骤自行构建。

## 首次启动：通知与登录启动

首次启动时，应用会询问是否在登录后自动运行。选择“启用”会注册登录启动；选择“Not Now”不会修改登录项，之后仍可在设置中更改。

通知权限不会自动弹出请求。只有在设置中明确点击启用通知后，应用才会请求 macOS 通知授权。拒绝后可前往“系统设置 > 通知”调整权限。

## 数据源健康

正常情况下，面板显示最近一次成功刷新后的数据。以下状态通常意味着数据源需要检查：

- 菜单栏显示 `Codex --`：尚未读到两个已知限额窗口，或还没有可用日志。
- 面板显示无数据：确认 Codex 已生成会话日志，并检查 `CODEX_HOME` 是否正确。
- 数据显示为陈旧或失败：应用保留最后一次成功快照，同时显示错误；可先点击“重试”。
- 新建的 `sessions` 或 `archived_sessions` 目录会由应用定期恢复发现，无需重新安装。

## 重建索引

在面板中点击“重建”，应用会清空本地 SQLite 汇总索引并重新扫描所有当前及已归档会话。此操作不会修改或删除源 JSONL 日志。

适合重建的情况包括：项目统计明显不一致、日志被替换或移动后没有更新，以及升级后本地索引无法正常读取。数据量较大时重建可能需要一些时间。

## 故障排查

- **没有菜单栏图标**：确认系统为 macOS 14 或更高版本，并从“活动监视器”检查应用是否正在运行。
- **没有用量数据**：运行 Codex 产生至少一条会话记录，确认 `CODEX_HOME` 或默认 `~/.codex` 目录存在。
- **限额一直缺失**：限额只会在 Codex 日志包含 rate-limit 观测时出现；先完成一次产生 Token 计数的会话，再重试扫描。
- **数据不刷新**：点击“重试”；仍无效时执行“重建”。
- **应用打不开**：当前本地构建采用 ad-hoc 签名且未公证。请确认产物来自可信来源，并使用 Finder 的“打开”操作查看系统提示。
- **构建失败**：确认 Swift 工具链可用，运行 `swift --version`，随后分别执行 `swift test` 与 `bash Scripts/build-app.sh` 定位阶段。

## 已知的格式兼容风险

本项目依赖 Codex 本地 JSONL 日志的现有字段结构。该结构不是本项目控制的稳定公共接口；Codex 更新可能新增、重命名或移除字段，导致部分事件被安全忽略、限额暂时缺失或统计停止更新。遇到兼容问题时请保留隐私脱敏后的最小结构样例并提交 issue，不要上传真实提示词、回复、凭据或完整会话日志。
